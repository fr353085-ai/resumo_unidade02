# resumo_unidade02
A ideia nesta Unidade II é mergulharmos de cabeça na programação assistida, mas com um olhar bem atento: o objetivo aqui não é apenas usar a IA para cuspir código, e sim aprender a construir uma parceria real e estratégica com ela. Hoje em dia, como mostram os dados de instituições como McKinsey e FGV, saber operar uma IA não é mais um luxo, é sobrevivência profissional. Ela impacta diretamente a nossa produtividade e até nossa qualidade de vida, desde que a gente saiba manter o controle da situação.
Para que essa colaboração funcione sem virar uma bagunça, a gente precisa adotar um workflow sustentável. Isso significa que você continua sendo o arquiteto: você planeja a lógica, usa ferramentas como o Copilot ou Replit para implementar, mas nunca abre mão de revisar e testar cada linha. Existe um peso ético e legal nisso que não podemos ignorar, envolvendo desde a segurança dos dados e a LGPD até o risco de aceitar um código que venha com vícios ou licenças problemáticas. No fim do dia, a responsabilidade pelo que vai para produção é 100% humana.
Ao analisarmos casos reais de equipes que já adotaram esses "copilotos", fica claro que o ganho de velocidade é absurdo, mas a dependência cega é um perigo real. Por isso, nosso debate central é sobre o limite da confiança: até onde a máquina ajuda e onde ela começa a nos "emburrecer"? Para fechar essa etapa, cada um vai elaborar um parecer sincero sobre o que praticou, relacionando a mão na massa com essa base teórica de boas práticas e ética que discutimos. A mensagem principal é que a IA é uma parceira incrível para lidar com o trabalho pesado, deixando a parte criativa e a tomada de decisão com a gente. Com essa mentalidade de "piloto", estaremos prontos para explorar o universo do Low Code e No Code na próxima unidade.
